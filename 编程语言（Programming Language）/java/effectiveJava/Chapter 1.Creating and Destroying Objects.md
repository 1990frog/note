# 用静态工厂方法代替构造器
## 优点
1. 静态方法有名称，可以准确的描述返回的类型，产生的客户端代码更易于阅读
2. 不必每次调用都创建一个新对象
3. 可以返回原返回的类型的任何子类型的对象（适用于基于接口的框架）

# 遇到多个构造器参数时要考虑使用构建器
使用重叠构造器，当有许多参数的时候，客户端代码会很难编写，并且难以阅读
使用javabean模式，无法保证类的不可变性
Builder模式，可以保证类似重叠构造器那样的安全性，也能保证javabean类似的可读性
Builder模式模拟了具名的可选参数
